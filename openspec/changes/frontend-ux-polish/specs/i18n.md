# Delta: i18n — Day Abbreviation Keys

## Domain: i18n

### Requirement: Day abbreviation keys MUST exist in all supported locales

The i18n files for English, Spanish, and French SHALL contain translation keys for weekday abbreviations used by the daily spending chart.

#### Scenario: English locale has day keys

- GIVEN the English i18n file (`en.json`)
- WHEN the `analytics.days` section is queried
- THEN it SHALL contain keys `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`
- AND values SHALL be "Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"

#### Scenario: Spanish locale has day keys

- GIVEN the Spanish i18n file (`es.json`)
- WHEN the `analytics.days` section is queried
- THEN it SHALL contain keys `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`
- AND values SHALL be "Lun", "Mar", "Mié", "Jue", "Vie", "Sáb", "Dom"

#### Scenario: French locale has day keys

- GIVEN the French i18n file (`fr.json`)
- WHEN the `analytics.days` section is queried
- THEN it SHALL contain keys `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`
- AND values SHALL be "Lun", "Mar", "Mer", "Jeu", "Ven", "Sam", "Dim"
